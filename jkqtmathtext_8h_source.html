50/*! \brief this class parses a mathematical markup string and can then draw the contained text/equation onto a <a href="http://doc.qt.io/qt-5/qpainter.html">QPainter</a>
51 \ingroup jkqtmathtext_render
52
53
54 JKQTMathText is a self-contained mathematical markup renderer for Qt. It is used to renderer
55 labels in JKQTPlotter/JKQTBasePlotter, but can be used independently.
56 The class does not depend on any library, except Qt.
57
58 The implementation is split over several classes:
59 - JKQTMathText implements drawing, based on a memory representatioj of the markup. It also provides an interface for class users.
60 - JKQTMathTextParser and its children like JKQTMathTextLatexParser parses mathamtical markup.
61 - The nodes summarized in \ref jkqtmathtext_items are used to build the memory representation of the markup.
62 .
63
64 In particular JKQTMathTextLatexParser actually parses e.g. a LaTeX string and draws it in pure C++. It does NOT rely
65 on an installed LaTeX for the rendering! See \ref jkqtmathtext_supportedlatex for a description of the supported LaTeX subset.
66
67 More generally these pages describe how the class renders math markup:
68 - \ref jkqtmathtext_renderingmodel for a description of the rendering model of JKQTMathTextLatexParser
69 - \ref jkqtmathtext_fonthandling for a description of the font handling in JKQTMathTextLatexParser
70 (which is modelled after LaTeX, but with tweaks especially for use in the context of GUI code)
71 - \ref jkqtmathtext_supportedlatex for a description of the supported LaTeX subset
72 .
73
74 \section JKQTMathTextUsageThreadSafety Thread-Safety/Re-Entrancy Guarantees
75 All functions in this class are reentrant. Therefore different instances off JKQTMathtext can be used in parallel in different threads.
76 DO NOT however use the same instance from multiple threads, as the functions are not thread-safe!
77
78 Also note that there are certain caches that reuse information (e.g. about fonts) from previous runs, which allow different instances
79 (also over different threads) to profit from each other. On the other hand, this strategy requires a bit of overhead due to synchronization,
80 but usually the speedup outweighs the overhead significantly!

83 \section JKQTMathTextUsage Usage
84
85 \subsection JKQTMathTextUsageParsing Parsing Functions
86 The class provieds two flavours of a parsing function:
87 - There is a function \c parse(markup, markupType) that accepts the markup, the type of markup (as enum DefaultParserTypes) and parser options.
88 This function determines the appropriate parser class, based on the markup type.
89 - A templated function parse<TParser>(markup, options) that gets a parser class to be used on the markup.
90 .

92 \subsection JKQTMathTextUsageDirect Drawing Functions
93 The class provides different variants of the drawing function draw() that paints using an
94 externlly provided <a href="https://doc.qt.io/qt-6/qpainter.html">QPainter</a>. These variants
95 either paint into a QRect, or starting from a single location (x,y).
96
97 The QRect-variants can align the render result inside the rectangle, whereas the
98 location-variants draw from a position that is on the left-hand side of the output's baseline:
99
100 \image html jkqtmathtext/jkqtmathtext_node_geo.png

102 Note that you first need to call one of the \ref JKQTMathTextUsageParsing so there is anything to render!

133 \subsection JKQTMathTextErrorHandling Error Handling
134
135 The class is designed to be as robust as possible and will still return some output, even if the equation contains some errors.
136 Nevertheless, several errors are detected while parsing. You can get a list of error messages using getErrorList() after calling parse().
137 Also parse() will return \c false if an error occured while parsing.

139 \subsection JKQTMathTextUsageExample Example Code
140 This small piece of C++ code may serve as an example of the usage and capabilities of the class:
141 \code
142 // create a JKQTMathText object.
143 JKQTMathText mathText;
144
145 // configure its properties to influence the rendering (e.g. fonts to use, font size, ...)
146 mathText.useXITS();
147 mathText.setFontSize(20);
148
149 // parse some LaTeX code (the Schroedinger's equation)
150 mathText.parse("$\\left[-\\frac{\\hbar^2}{2m}\\frac{\\partial^2}{\\partial x^2}+V(x)\\right]\\Psi(x)=\\mathrm{i}\\hbar\\frac{\\partial}{\\partial t}\\Psi(x)$");
151
152 // draw the result into a QPixmap
153 QPixmap result=mathText.drawIntoPixmap();
154 \endcode
155
156 Alternatively you can also use this class with a <a href="https://doc.qt.io/qt-6/qpainter.html">QPainter</a>:
157
158 \code
159 // use the draw() methods to draw the equation using a QPainter (here onto a QPixmap)
160 QPainter painter;
161
162 // first we determine the size of the render output:
163 const JKQTMathTextNodeSize size=getSizeDetail(painter);
164
165 // now we can us that size information to render:
166 mathText.draw(painter, Qt::AlignCenter, QRectF(0,0,pix.width(), pix.height()), false);
167 \endcode

221 /** \brief parse the given math \a markup string with a parser derived from \a markupType.
222 *
223 * \param markup the string of math markup
224 * \param markupType defines the language the \a markup is written in (and is used to derive the parser to use)
225 * \param options Options for parsing, \see ParseOptions

227 * \returns \c true on success.

230 /** \brief parse the given math \a markup string, using the given parser class \a TParser
231 *
232 * \tparam TParser the parser (deived from JKQTMathTextParser) to be used
233 * \param markup the string of math markup
234 * \param options Options for parsing, \see ParseOptions
235 *
236 * \returns \c true on success.
237 */
238 template <class TParser>
239 inline bool parse(const QString &markup, ParseOptions options=DefaultParseOptions) {
240 static_assert(std::is_base_of<JKQTMathTextParser, TParser>::value, "in parse<TParser>() the type TParser has to be derived from JKQTMathTextParser to work!");
241 std::unique_ptr<TParser> p=std::unique_ptr<TParser>(new TParser(this));
242 if (parsedNode) delete parsedNode;
243 parsedNode=nullptr;
244 clearErrorList();
245 parsedNode=p->parse(markup, options);
246 return parsedNode!=nullptr;
247 }
